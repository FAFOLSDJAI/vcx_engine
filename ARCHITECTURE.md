# 语音通讯软件架构设计

## 技术栈选择
- **客户端**: Electron + React + TypeScript
- **实时通讯**: WebRTC (去中心化点对点)
- **信令服务器**: Node.js + Socket.IO (用于连接建立，不传输音视频)
- **数据持久化**: IndexedDB + SQLite

## 架构设计

### 1. 去中心化通信架构
- 使用 **WebRTC** 实现点对点音视频传输
- 使用 **信令服务器** 协助 SDP 交换和 ICE 候选者
- 语音数据不经过中央服务器，直接在用户间传输
- 采用 Mesh 或 SFU 架构支持多人语音

### 2. 核心模块

#### 用户系统模块
- 用户注册/登录
- 用户资料管理
- 在线状态追踪
- 好友系统

#### 服务器/频道管理模块
- 服务器创建和管理
- 频道（文本/语音）创建
- 权限系统（管理员、普通成员等）
- 邀请码机制

#### 语音通讯模块
- 音频采集和播放
- WebRTC 连接管理
- 音量检测和噪音抑制
- 静音/取消静音功能

#### 文本聊天模块
- 实时消息传输
- 消息历史记录
- @提及和表情符号
- 消息编辑和删除

## 技术选型细节

### 客户端
- **Electron**: 跨平台桌面应用
- **React 18**: UI 框架
- **TypeScript**: 静态类型检查
- **Tailwind CSS**: 样式设计
- **simple-peer**: WebRTC 封装库

### 信令服务器
- **Node.js**: 运行时
- **Socket.IO**: WebSocket 通信
- **Express**: HTTP 服务器

### 数据存储
- **IndexedDB**: 浏览器本地存储
- **SQLite**: 跨会话数据持久化
